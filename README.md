# 코루틴과 스레드의 차이

코루틴과 스레드의 관계?  
코틀린의 실행 단위는 코루틴이고 자바는 스레드라는데,  
코틀린은 결국 JVM 위에서 돌아가잖아.

1. **1:N 관계 (작업과 일꾼)**  
   **스레드 (일꾼)**: 운영체제가 관리하는 실제 리소스입니다. 생성 비용이 크고 메모리를 많이 차지합니다.  
   **코루틴 (작업)**: 스레드 위에서 돌아가는 작은 코드 조각입니다. 하나의 스레드에서 수천 개의 코루틴을 번갈아 가며 실행할 수 있습니다.


2. "멈춤(Suspend)"의 차이  
   코루틴의 마법은 여기서 나옵니다.  
   **스레드**: 네트워크 응답을 기다릴 때 스레드 자체가 차단(Blocking)되어 아무것도 못 하고 쉽니다. (비효율적)  
   **코루틴**: 응답을 기다려야 하면 실행 중이던 코루틴은 일시 중단(Suspend)되고, 그 스레드는 다른 코루틴을 처리하러 떠납니다. 응답이 오면 다시 비어있는 스레드가 해당 코루틴을 이어받아 실행합니다.


3. **왜 코루틴이 더 효율적인가?**  
   **Context Switch 비용**: 스레드를 바꿀 때는 운영체제 레벨에서 무거운 작업이 일어나지만, 코루틴을 바꿀 때는 단순히 JVM 내부에서 객체 포인터만 바꾸는 수준이라 훨씬 빠릅니다.  
   **메모리**: 스레드는 각각 약 1MB의 스택 메모리를 갖지만, 코루틴은 수 KB의 객체에 불과합니다.
